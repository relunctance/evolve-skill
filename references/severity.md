# 严重程度体系（P0-P3 + auto-severity）

## P0-P3 权重

| 标签 | 权重 | 说明 |
|------|------|------|
| P0 | 4 | 关键 — 功能完全失效 |
| P1 | 3 | 主要 — 功能受影响 |
| P2 | 2 | 中等 — 体验较差 |
| P3 | 1 | 轻微 — 改进建议 |

## 分类权重

| 分类 | 权重 | 场景 |
|------|------|------|
| functional | 2.0 | API/CLI/数据流错误 |
| performance | 1.5 | 性能瓶颈/超时 |
| layout | 1.2 | 排版/层级/坐标错位 |
| visual | 1.1 | 颜色/对比度/水印 |
| ux | 1.0 | 易用性/文档缺失 |
| other | 1.0 | 其他 |

## 判定规则

- 问题没有 `[Px]` 标签 → scanner 跳过
- 标签缺失视为最低优先
- 多个 Px 同名问题 → 累加
